# トラブルシューティング & FAQ

[English](TROUBLESHOOTING.md) · **日本語**

収集が失敗する・0件になる・本文が乱れる——そんな時の実用ガイドです。原因の
大半は「フィードURLの移転/失効」「サイトのボット拒否」「アプリの再起動が必要」
のいずれかで、たいてい1分で直せます。

> 収集トラブルのほぼ全てで、まず health check を実行してください。
> ```
> python cli.py verify              # 全源
> python cli.py verify --source <id>   # 1源だけ
> ```
> 各フィードに接続して状態を表示し、`verify_report.txt` に保存します。

---

## `verify` のステータスの読み方

| status | 意味 | 対処 |
|---|---|---|
| `ok` | 取得・解析に成功 | 何もしなくてOK |
| `empty` | 接続できたが0件 | URLは応答するがフィードでなくなっている。現行のRSS URLを探すか `html` に切替 |
| `robots_blocked` | robots.txt が取得を禁止 | `fetch_method: manual`（手動貼付）に切替、または別フィードを探す |
| `fetch_error` | 接続失敗（404/タイムアウト/DNS/403） | フィードURLが移転/停止。サイトを開いて新URLに更新、または `html`/`manual` に切替 |
| `parse_error` | RSS/Atom でない（多くはHTMLページ） | フィードでなくページを指している。正しいフィードURLにするか `fetch_method: html` |
| `manual` | 手動貼付源（自動取得しない） | 正常。「手で貼る」欄から追加 |

## よくある問題

| 症状 | 考えられる原因 | 対処 |
|---|---|---|
| **何も収集されない（0 new）** | 既に保存済み（重複排除）、またはテーマ絞り込みで隠れている | 再実行時は正常。テーマ絞りを外して全件表示、または別の国タブを確認 |
| **ある源が `fetch_error`** | フィードURLが変わった/サイト停止 | `verify` 実行→サイトを開いて `sources/*.yaml` の `url` を現行RSSに更新、または `html`/`manual` に切替 |
| **ある源が `empty` / `parse_error`** | URLがフィードでなく通常のWebページ | ページ内のRSSリンクを探す（`CONTRIBUTING.md` のウォークスルー参照）か、`fetch_method: html` で本文抽出 |
| **ある源が `robots_blocked`** | サイトが robots.txt でボットを拒否 | `fetch_method: manual` にして要約/引用を手で貼る（公正利用の範囲で） |
| **「記事本文まで取得」しても要約のまま** | 記事ページがボット拒否、またはJavaScript描画で本文を取得できない | 安全なフォールバック。理由はCLI出力とWeb UIの「取得メモ(Fetch notes)」に表示。`pip install trafilatura` で抽出精度が向上 |
| **本文がクッキー同意文/メニューだらけ** | ページに綺麗な記事領域が無く、定型文を拾った | その源は `--full` をやめて要約を使う、または `manual` に切替。抽出品質はサイト依存 |
| **古い画面が出る/編集が反映されない** | 旧 `python app.py`（やインストール版）がポート5000を握っている、またはFlaskが自動リロードしない | 旧プロセスを止めてソースから再起動。編集中のライブ反映は `OPENBEAT_DEBUG=1` で起動 |
| **「ポート5000が使用中」** | 別プロセス/前回起動がポートを占有 | 旧ウィンドウを閉じる、または別ポート指定 `OPENBEAT_PORT=5050 python app.py` |
| **テーマ絞りで0件** | キーワードが記事の言語/表記と一致しない | テーマは単純なテキスト検索。`themes/<name>.yaml` にキーワード追加（源の言語＝日本語/仏語も入れる） |
| **UIが英語で日本語にしたい** | 言語は環境変数で制御 | 起動前に `OPENBEAT_LANG=ja`（PowerShell: `$env:OPENBEAT_LANG="ja"`、Mac/Linux: `export OPENBEAT_LANG=ja`） |
| **macOS「開発元を検証できません/マルウェアでないか確認できません」** | アプリが未署名（公証は予定） | アプリを右クリック→**開く**、または システム設定→プライバシーとセキュリティ→**このまま開く**、もしくは `xattr -dr com.apple.quarantine <app>` |
| **Windows SmartScreen の警告** | インストーラが未署名 | **詳細情報→実行** をクリック（コード署名は予定） |
| **データベースを初期化したい** | 古い/テストのアイテムが一覧に残っている | アプリを止めて `data/rwt.sqlite` を削除し、再収集 |

## FAQ

**私のデータを使ったり、クラウドAIに何か送信したりしますか？**
いいえ。収集ツールは**あなたのPC内だけ**で動作します。公開された
RSS/HTML/CSV を取得してローカルの SQLite に保存するだけで、AIサービスに何かを
送信することは一切なく、アカウントやAPIキーも不要です。（AIによる分類・分析は、
別途提供の有料「クローズ層」の機能で、本ツールには含まれません。）

**収集データはどこに保存されますか？**
アプリの隣のローカル SQLite ファイル `data/rwt.sqlite` です。書き出しは
`exports/`（および「生コーパスJSONを書き出し」で保存するJSON）に出ます。

**源のURLを変えました。動くか確認するには？**
`python cli.py verify --source <id>` を実行。`ok` なら取得・解析OK。
`fetch_error`/`empty`/`parse_error` の場合は上の表に従ってください。実践例は
`CONTRIBUTING.md`（「自分の源を1から足す」）にあります。

**なぜ全文でなく要約だけの記事があるのですか？**
既定では見出し＋要約のみ保存します（軽量・丁寧）。「記事本文まで取得」（または
`--full`）を有効にすると、サイトが許す範囲で本文を取得します。ボット拒否や
JavaScript描画のページは要約にフォールバックします。

**自動で収集し続けるには？**
`run_scheduled.py` を cron（Linux/macOS）や タスクスケジューラ（Windows）で。
`README.md` の「Scheduled crawling」を参照。

**自分の国・媒体・トピックを足せますか？**
はい。`sources/*.yaml`（情報源）や `themes/*.yaml`（トピック）を編集するだけ。
コード不要です。手順は `CONTRIBUTING.md` のウォークスルーを参照。

**この情報収集は合法ですか？**
本ツールは公開情報のみを収集し、robots.txt と各サイトの規約を尊重し、全アイテム
に出典リンクを付け、要約を保存します（全文転載はしません）。各サイトの規約順守は
利用者の責任で、想定用途は出典明記つきの要約/引用です。

**それでも動かない/バグを見つけた場合は？**
`verify_report.txt` の出力と、実行コマンドまたは画面を添えて GitHub に issue を
立ててください。源またはツールの修正を手伝います。
