# OpenBeat Collector — ロードマップ

[English](ROADMAP.md) · **日本語**

地政学ショック × 中小企業の公開情報を **収集 → 分類 → 分析 → 各国レポート化 → 個社診断** する
オープンコア製品のロードマップです。

- **オープン側（無料・OSS）= 収集まで**：`OpenBeat_Collector`
- **クローズ側（有料）= 収集の後すべて**：`rwt-closed`
- **境界**：Collector 直後。受け渡し物は生コーパス JSON（`schema: rwt.raw_corpus` v1.0）。

オープン側はコミュニティ資産（情報源レジストリ・収集の堅牢化）として広く使われることを狙い、
価値の核（分類・分析・各国比較・診断・一次データ）はクローズ側で提供します。

## 設計上の鉄則（全レイヤー共通）

- **出典トレーサビリティ**：すべての主張・判定は出典（URL）に紐づく。紐づかない主張は出さない（幻覚対策）。
- **Human-in-the-loop**：生成物は常に `status: draft`。公開・配信の前に人間が編集・承認する。
- **再現性**：情報源・タグ辞書・プロンプトはリポジトリ管理。
- **プライバシー優先**：一次データは既定で匿名化。蓄積はオプトイン。個人情報は扱わない。
- **中立性**：地政学テーマゆえ、断定を避け出典の提示に徹する。

### ビート・パック マーケットプレイス（オープン側の community moat）

「テーマ適合」をテンプレ化し、**各ジャーナリストの専門領域（ビート）を1つの"パック"として共有・取り込みできる**仕組み。
収集アセット（テーマ＋情報源）はオープン、精緻な分類辞書・分析プロンプトはクローズ層（有料）に置き、オープンコア境界を保つ。

- **パックの単位**：`manifest.yaml`（id / 領域名 / 作者 / 言語 / 版 / ライセンス / 信頼メモ）＋ `theme.yaml`（検索キーワード・多言語）＋ `sources.yaml`（厳選情報源）＋（任意・クローズ）`tags.yaml`（分類辞書）。
- **既存資産の延長**：`themes/*.yaml`・`sources/*.yaml`（コミュニティ資産）と、アプリの `OPENBEAT_SOURCES_DIR` / `OPENBEAT_THEMES_DIR` / `OPENBEAT_USER_SOURCES_DIR`（ユーザー追加オーバーレイ）・UIの情報源管理・`CONTRIBUTING` の PR モデルの上に構築できる。
- **想定実装（小）**：(1) パック manifest 形式の定義、(2) `cli.py pack install / list / validate`（既存 `validate`・`verify` を流用）、(3) コミュニティ・パック集リポジトリ＋索引 `packs.json` → アプリから 1 クリック取り込み。
- **信頼ティア**：「コミュニティ・パック」と「検証済みパック」（編集機関・専門団体の監修）を分離。`verify` でパック健全性バッジ（フィードの腐敗検知）。
- **例**：科学・技術／食品・食料／医療・公衆衛生／気候・エネルギー（4ビートパック実装済み）。
- **緊急パック（Rapid-Response）**：世界的危機の発生時に専門家監修の信頼ソースを世界同時配布。`type: emergency`・`trust_tier: federation-certified`・会員/一般価格。`packs/crisis-response` 実装済み（WHO/ReliefWeb/UN News/GDACS/IFRC）。中核商品。
- **ネットワーク効果**：記者が増える→パックが増える→価値が上がる→記者が増える。クローズ側の一次データ堀と両輪。

### 事業モデル：連盟OEM・3製品スイート

本ツールは製品スイート **OpenBeat**（Open＝オープンコア無償本体／Beat＝記者の専門領域＝パック）の一部。世界科学ジャーナリスト連盟（WFSJ・約67の国別団体）の各加盟団体が、自団体の資金獲得のために自団体ブランドで提供する **OEM 製品** として位置づける。製品は4つ：① **OpenBeat Collector**（本リポ・無償OSS＋連盟認定の有料ソースパック）② **OpenBeat Newsroom**（`rwt-closed`・編集配信・有料）③ **OpenBeat Media**（`rwt-media`・新規＝ジャーナリスト自身の蓄積型メディア・会員制・週刊ニュースレター・有料購読。Newsroomから配信）④ **OpenBeat Market**（`rwt-store`・パック/研修販売）。**収益2軸**＝記者側（パック/研修/Newsroom＝団体資金）＋読者側（Media購読＝オーディエンス収益）。認定パック・緊急パックは **会員価格／一般価格** で販売し、会員特典＝加盟動機として会員拡大につなげる。詳細は事業モデル文書（OneDrive `OpenBeat_事業モデル_日英.docx`）・`OpenBeat_Newsroom_v2_仕様.md`・`OpenBeat_Media_仕様.md` 参照。※「OpenBeat（Toolkit）」は旧称＝出発点の一機能。リポジトリ名は据え置き、表示・ブランドは OpenBeat に統一。

凡例：- ✅ パック manifest 形式（`packs/SCHEMA.md`）と4パイロットパック ・ ✅ パックストアUI（サーバ配信カタログ＋無料/有料バッジ＋購入導線＋ライセンスキー解錠＋ティア制限 無料1/Pro複数）`rwt/packs.py`・`templates/store.html`（`tests/test_store.py` 12項目） ・ ✅ `cli.py pack install/list/validate/activate/license` ・ 🔜 検証済みパックへのクローズ層 tags.yaml 付与 ・ 🔜 コミュニティ投稿フロー（PR）と健全性バッジ

### クローズ側（収集の後すべて・有料）

- ✅ 取込（生コーパス JSON）→ 分類（ルール＋任意 LLM・共通タグスキーマ）
- ✅ 分析（要約／日本への示唆／早期シグナル・**出典必須**）
- ✅ 国別カード ＋ 横断ダイジェスト（**国際比較**：同一ショックへの各国対応の差分・重複排除統合）
- ✅ 編集ダッシュボード（取込→編集→承認 draft/reviewed/published→時系列）
- ✅ 英語の海外寄稿ドラフト生成（出典必須）
- ✅ 中小企業向け研修メニュー生成（ケース＋設問＋アクション・出典必須）
- ✅ **AI 診断士**（企業個別の事業継続リスク診断・証拠駆動スコア・出典必須）
- ✅ 配信ツール（email / newsletter HTML / teaser・draft 配信ガード）＝v1（整形のみ・実送信なし・テキスト中心）
- 🔜 **OpenBeat Newsroom v2（脱テキスト＋自動配信）**：マルチフォーマット（図表/インフォグラフィック・音声・動画）＋多言語自動出力＋自動マルチチャネル配信（メール実送信/Web・CMS/RSS/SNS/Webhook/push）＋**緊急ラピッドレスポンス配信**（危機パック認定→全団体・全言語・全チャネル一斉）。出典・Human-in-the-loop を全フォーマットで維持。詳細は OneDrive `OpenBeat_Newsroom_v2_仕様.md`。段階：v2.0 配信コネクタ＋可視化／v2.1 多言語＋SNS／v2.2 音声＋push＋緊急SLA／v2.3 動画
- ✅ **一次データ基盤 ＋ 業界ベンチマーク**（診断インテークの匿名蓄積→業界別集計→個社比較）
- ✅ **ビート別の有料分析層**（`rwt-closed/packs/<id>/` に科学技術・食品・医療・気候の分類辞書 tags.yaml＝5カテゴリ schema。`--pack <id>` / `--pack-dir` で分類→分析→カード/研修/寄稿/診断をビート別に実行。オープン側パックと同 id で対応・ストア購入＝クローズ層解錠）
- 🔜 LLM プロバイダ拡充（ローカル / OSS モデル対応）
- 🔜 多言語 UI・レポートの英語/他言語出力の拡充
